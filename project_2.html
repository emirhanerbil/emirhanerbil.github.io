About Project

- Sometimes fine-tuning a chatbot according to our own questions can be costly and time-consuming.

- Then the rag approach comes into play.

- Here, instead of fine-tuning the entire llm, we give the relevant paragraphs from our information source to llm and ask it to answer this question in this context.

- In order to understand which paragraphs are relevant, we divide the entire pdf into pieces called chunks.

- Then, we convert these chunks to vectors using the embedding model and save them to the database if we want.

- After converting the question asked to vectors in the same way, we do vector search between the chunk vectors and question vectors.

- We get the best 3 results and send these chunks to llm and have it answer the question.

- My llm preference was llama3.

- Because those who do this project usually use Open Ai or some paid vector search sites.

- Since I wanted what I did to work locally and completely free, I preferred llama3 as llm and bge-m3, which is found in huggingface, as the embedding model.
